# <a name="cmfcvisualmanager-class"></a>CMFCVisualManager – třída

Poskytuje podporu pro změnu vzhledu aplikace na globální úrovni. Třída `CMFCVisualManager` pracuje společně s třídou, která poskytuje pokyny k nakreslení ovládacích prvků GRAFICKÉho uživatelského rozhraní aplikace pomocí konzistentního stylu. Tyto další třídy jsou označovány jako `CMFCBaseVisualManager`vizuální manažeři a dědí z .

Další podrobnosti naleznete ve zdrojovém kódu umístěném ve složce **MFC\\knihovny\\VC src\\** instalace sady Visual Studio.

## <a name="syntax"></a>Syntaxe

```
class CMFCVisualManager : public CMFCBaseVisualManager
```

## <a name="remarks"></a>Poznámky

Vzhledem `CMFCVisualManager` k tomu, že třída řídí grafické uživatelské rozhraní `CMFCVisualManager`aplikace, každá aplikace může `CMFCVisualManager`mít buď jednu instanci nebo jednu instanci třídy odvozené z . Vaše aplikace může také `CMFCVisualManager`fungovat bez . Pomocí statické `GetInstance` metody získat ukazatel na `CMFCVisualManager`aktuální odvozený objekt.

Chcete-li změnit vzhled aplikace, musíte použít jiné třídy, které poskytují metody pro kreslení všech vizuálních prvků aplikace. Některé příklady těchto tříd jsou [TŘÍDA CMFCVisualManagerOfficeXP , TŘÍDA](../../mfc/reference/cmfcvisualmanagerofficexp-class.md) [CMFCVisualManagerOffice2003](../../mfc/reference/cmfcvisualmanageroffice2003-class.md)a [TŘÍDA CMFCVisualManagerOffice2007](../../mfc/reference/cmfcvisualmanageroffice2007-class.md). Pokud chcete změnit vzhled aplikace, předejte jeden z těchto `SetDefaultManager`vizuálních manažerů do metody . Příklad, který ukazuje, jak může aplikace napodobovat vzhled sady Microsoft Office 2003, naleznete v [tématu CMFCVisualManagerOffice2003 Class](../../mfc/reference/cmfcvisualmanageroffice2003-class.md).

Všechny metody kreslení jsou virtuální. To umožňuje vytvořit vlastní vizuální styl pro grafické uživatelské rozhraní aplikace. Pokud chcete vytvořit vlastní vizuální styl, odvodit třídu z jedné z tříd vizuálního manažera a přepsat metody kreslení, které chcete změnit.

## <a name="example"></a>Příklad

Tato ukázka ukazuje, jak vytvořit konkretizovat standardní a vlastní `CMFCVisualManager` objekty.

```cpp
void CMFCSkinsApp::SetSkin (int iIndex)
{   // destroy the current visual manager
    if (CMFCVisualManager::GetInstance () != NULL)
    {
        delete CMFCVisualManager::GetInstance ();
    }
    switch (iIndex)
    {
    case 0:
        CMFCVisualManager::GetInstance ();
        // create the standard visual manager
        break;
    case 1:
        new CMyVisualManager ();
        // create the first custom visual manager
        break;
    case 2:
        new CMacStyle ();
        // create the second custom visual manager
        break;
    }
    // access the manager and set it properly
    CMFCVisualManager::GetInstance ()->SetLook2000 ();
    CMFCVisualManager::GetInstance ()->RedrawAll ();
}
```
