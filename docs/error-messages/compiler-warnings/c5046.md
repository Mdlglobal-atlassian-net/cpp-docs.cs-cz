# <a name="compiler-warning-level-2-c5046"></a>Kompilátor C5046 upozornění (úroveň 2)

> "*funkce*": Symbol, zahrnující typ s vnitřním propojením nedefinovaná.

## <a name="remarks"></a>Poznámky

Kompilátor zjistil použití funkce, která nemá definici, ale signatura této funkce zahrnuje typy, které nejsou viditelné mimo této jednotce překladu. Vzhledem k tomu, že tyto typy nejsou externě viditelné, žádné jiné jednotce překladu může poskytnout definici pro tuto funkci tak program nemůže být propojení se úspěšně vytvořilo.

Typy, které nejsou viditelné mezi jednotkami překladu patří:

- Typy deklarované v anonymním oboru názvů.

- Místní a nepojmenované třídy

- Specializace šablon, které používají tyto typy jako argumenty šablony.

Toto upozornění je nového v sadě Visual Studio 2017 verze 15.8.

## <a name="example"></a>Příklad

Tento příklad ukazuje dva C5046 upozornění:

```cpp
// C5046p.cpp
// compile with: cl /c /W2 C5046p.cpp

namespace {
    struct S {
        // S::f is inside an anonymous namespace and cannot be defined outside
        // of this file. If used, it must be defined somewhere in this file.
        int f();
    };
}

// g has a pointer to an unnamed struct as a parameter type. This type is
// distinct from any similar type declared in other files, so this function
// cannot be defined in any other file.
// Note that if the struct was declared "typedef struct { int x; int y; } S, *PS;"
// it would have a "typedef name for linkage purposes" and g could be defined
// in another file that provides a compatible definition for S.

typedef struct { int x; int y; } *PS;
int g(PS p);

int main()
{
    S s;
    s.f();      // C5046 f is undefined and can't be defined in another file.
    g(nullptr); // C5046 g is undefined and can't be defined in another file.
}
```

Chcete-li vyřešit tyto problémy, definování funkcí v tomto souboru.
