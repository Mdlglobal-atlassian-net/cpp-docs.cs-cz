# <a name="compiler-warning-level-1-c4477"></a>Kompilátor upozornění (úroveň 1) C4477

> "*funkce*': řetězec formátu"*řetězec*"vyžaduje argument typu"*typ*", ale variadický argument *číslo* má typ"*typ*.

Kompilátor zjistil nesoulad mezi typ argumentu, které jsou nutné k uspokojení zástupný symbol ve formátovacím řetězci a typ zadaný argument. Správné použití printf a scanf rodiny variadické funkce vyžaduje, abyste zadali argumenty nebo více typů určených ve formátovacím řetězci. Neshoda obecně znamená, že je chyba v kódu.

Informace pro argumenty přidružené k zástupné symboly printf řady formát řetězce najdete v tématu [syntaxe specifikace formátu: funkce printf a wprintf](../../c-runtime-library/format-specification-syntax-printf-and-wprintf-functions.md). Najdete v dokumentaci pro informace specifické pro funkce *funkce*.

Toto upozornění je nového v sadě Visual Studio 2015.

## <a name="example"></a>Příklad

Tato ukázka zobrazuje dvě C4477 upozornění pro první printf_s – funkce, když dva argumenty jsou zjištěna nesprávného typu a také ukazuje, jak opravit problémy.

```cpp
// C4477p.cpp
// compile with: cl /c /W4 C4477p.cpp
#include <stdio.h>

void print_func(int a, float f)
{
    printf_s("This takes int and double arguments %d and %f.\n", f, a); // C4477, expected int then double
    printf_s("This takes int and double arguments %d and %f.\n", a, f); // fix: types in correct order
}
```

V první chybovou zprávu, kompilátor zjistí, že **double** je zadán argument při **int** se očekává. A **float** je automaticky povýšen na **double** v seznamu variadických argumentů. Druhá chybová zpráva znázorňuje, že kompilátor očekává **double** argument, ale **int** byl zadán. Kompilátor nemůže určit, že mají vyměnit pořadí argumentů. Musí zkontrolovat řetězec formátu pro určení pořadí skutečný argument a poskytnete správné typy argumentů.
