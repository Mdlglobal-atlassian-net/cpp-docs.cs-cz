# <a name="compiler-warning-c5045"></a>C5045 upozornění kompilátoru

> Kompilátor vloží zmírnění hrozby Spectre pro zatížení paměti, pokud přepínač/qspectre zadané

## <a name="remarks"></a>Poznámky

Upozornění umožňuje C5045 uvidíte, jaké vzory ve vašem kódu způsobit zmírnění hrozby Spectre, jako je například LFENCE, má být vložen při [/qspectre](../../build/reference/qspectre.md) je zadána možnost kompilátoru. To vám umožní identifikovat soubory kódu, které jsou ovlivněny potíže se zabezpečením. Toto upozornění je čistě informační: omezení rizik není vložen, dokud je znovu zkompilovat pomocí **/qspectre** přepnout. Funkce C5045 je nezávislá **/qspectre** přepnout, takže je lze využít oba ve stejné kompilaci.

Toto upozornění je nového v sadě Visual Studio 2017 verze 15.7 a je vypnuto ve výchozím nastavení. Použití [/Wall](../../build/reference/compiler-option-warning-level.md) povolit všechna upozornění, které jsou ve výchozím nastavení, vypnutí nebo __/w__*n*__5038__ umožňující C5045 jako úroveň *n* upozornění. V integrovaném vývojovém prostředí, je výchozí úroveň upozornění **w3** a toto upozornění se dá nastavit v projektu **stránky vlastností** dialogového okna. Otevřít **vlastnosti konfigurace** > **C/C++** > **příkazového řádku** a **další možnosti** Přidejte */w35045*, klikněte na tlačítko **OK**. Další informace najdete v tématu [upozornění kompilátoru, které jsou ve výchozím nastavení vypnuta](../../preprocessor/compiler-warnings-that-are-off-by-default.md). Informace o tom, jak zakázat varování podle verze kompilátoru najdete v tématu [upozornění kompilátoru podle verze kompilátoru](compiler-warnings-by-compiler-version.md).

## <a name="example"></a>Příklad

Následující příklad vyvolá upozornění C5045 při kompilaci pomocí sady Visual Studio 2017 verze 15.7 s **/Wall** nebo **/w35045** a **w3** možnosti:

```cpp
// C5045.cpp
// Compile with: cl /EHsc /W3 /w35045 C5045.cpp

int G, G1, G2;

__forceinline
int * bar(int **p, int i)
{
    return p[i];
}

__forceinline
void bar1(int ** p, int i)
{
    if (i < G1) {
        auto x = p[i]; // C5045: mitigation here
        G = *x;
    }
}

__forceinline
void foo(int * p)
{
    G = *p;
}

void baz(int ** p, int i)
{
    if (i < G1) {
        foo(bar(p, i + G2));
    }
    bar1(p, i);
}

int main() { }
```

Ve výstupu kompilátoru při upozornění je povolená vypadá přibližně takto:

```Output
C:\Users\username\source\repos\C5045>cl /W3 /w35045 C5045.cpp
Microsoft (R) C/C++ Optimizing Compiler Version 19.14.26431 for x86
Copyright (C) Microsoft Corporation.  All rights reserved.

C5045.cpp
c:\users\username\source\repos\c5045\c5045.cpp(16) : warning C5045: Compiler will insert Spectre mitigation for memory load if /Qspectre switch specified
c:\users\username\source\repos\c5045\c5045.cpp(15) : note: index 'i' range checked by comparison on this line
c:\users\username\source\repos\c5045\c5045.cpp(17) : note: feeds memory load on this line
Microsoft (R) Incremental Linker Version 14.14.26431.0
Copyright (C) Microsoft Corporation.  All rights reserved.

/out:C5045.exe
C5045.obj
```

Tyto zprávy zobrazit, že omezení rizik by byl vložen v řádku 16. Rovněž uvedena, že omezení rizik je potřeba proto, že na řádku 15 index i zatížení paměti kanálů na řádku 17. Spekulační se provádí přes panelu a bar1, ale je omezení rizik efektivní při umístění na řádku 16.
