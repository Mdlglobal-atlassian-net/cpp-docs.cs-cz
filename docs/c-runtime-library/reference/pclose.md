# <a name="_pclose"></a>_pclose

Čeká na nový procesor příkazů a zavře datový proud na přidruženém kanálu.

> [!IMPORTANT]
> Toto rozhraní API nelze použít v aplikacích, které se spouštějí v prostředí Windows Runtime. Další informace naleznete v tématu [funkce CRT, které nejsou podporovány v aplikacích univerzální platformy Windows](../../cppcx/crt-functions-not-supported-in-universal-windows-platform-apps.md).

## <a name="syntax"></a>Syntaxe

```C
int _pclose(
FILE *stream
);
```

### <a name="parameters"></a>Parametry

*Proudu*<br/>
Vrácená hodnota z předchozího volání **do _popen**.

## <a name="return-value"></a>Návratová hodnota

Vrátí stav ukončení ukončujícího příkazového procesoru nebo -1, pokud dojde k chybě. Formát vrácené hodnoty je stejný jako u **_cwait**, s výjimkou bajtů nižšího a nejvyššího řádu. Pokud je datový proud **NULL**, **_pclose** nastaví **errno** na **EINVAL** a vrátí -1.

Informace o těchto a dalších kódech chyb naleznete [v tématu _doserrno, errno, _sys_errlist a _sys_nerr](../../c-runtime-library/errno-doserrno-sys-errlist-and-sys-nerr.md).

## <a name="remarks"></a>Poznámky

Funkce **_pclose** vyhledá ID procesu příkazového procesoru (Cmd.exe) spuštěného přidruženým **_popen** voláním, provede [volání _cwait](cwait.md) na novém příkazovém procesoru a zavře datový proud na přidruženém kanálu.

Ve výchozím nastavení je globální stav této funkce vymezen na aplikaci. Chcete-li to změnit, naleznete [v tématu Globální stav v CRT](../global-state.md).
