## <a name="microsoft-specific"></a>Specifické pro Microsoft  
 V aplikaci Microsoft C++ jsou identifikátory dvě úvodní podtržítka vyhrazené pro implementace kompilátoru. Konvence Microsoft je proto předcházet klíčová slova specifická pro společnost Microsoft se dvojité podtržítka. Tato slova nelze použít jako názvy identifikátor.  
  
 Rozšíření Microsoft jsou ve výchozím nastavení povolené. Zkontrolujte, zda jsou plně přenositelné programy, můžete zakázat rozšíření Microsoft zadáním ANSI kompatibilní [/Za](../build/reference/za-ze-disable-language-extensions.md) možnost příkazového řádku (kompilace pro kompatibilitu ANSI) během kompilace. Když to uděláte, jsou zakázané klíčová slova specifická pro společnost Microsoft.  
  
 Pokud jsou povolené rozšíření Microsoft, můžete použít klíčová slova specifická pro společnost Microsoft ve vašich aplikacích. Pro kompatibilita se specifikací ANSI jsou uvedena tato klíčová slova ve dvojité podtržítko. Pro zpětnou kompatibilitu, podtržítko jedním verzích všech podtržena dvojitou klíčových slov s výjimkou `__except`, `__finally`, `__leave`, a `__try` jsou podporovány. Kromě toho `__cdecl` je k dispozici žádné úvodní podtržítka.  
  
 `__asm` – Klíčové slovo nahrazuje C++ `asm` syntaxe. `asm` je vyhrazené pro kompatibilitu s jiným implementacím C++, ale není implementováno. Použití `__asm`.  
  
 `__based` – Klíčové slovo má omezenou používá pro 32bitové a 64bitové verze cílového kompilace.
