**Specifické pro Microsoft**

V jazyce Microsoft C++ jsou identifikátory se dvěma předními podtržítky rezervovány pro implementace kompilátoru. Proto konvence Microsoftu předchází klíčová slova specifická pro společnost Microsoft s dvojitým podtržítkem. Tato slova nelze použít jako názvy identifikátorů.

Ve výchozím nastavení jsou rozšíření společnosti Microsoft povolena. Aby bylo zajištěno, že jsou vaše programy plně přenosné, můžete zakázat rozšíření společnosti Microsoft zadáním možnosti [/za \( Disable Language Extensions](../build/reference/za-ze-disable-language-extensions.md) v průběhu kompilace. Když to uděláte, některá klíčová slova specifická pro společnost Microsoft jsou zakázaná.

Pokud jsou rozšíření společnosti Microsoft povolena, můžete v programech použít klíčová slova specifická pro společnost Microsoft. V případě kompatibility se standardem ANSI jsou tato klíčová slova uvozena dvojitým podtržítkem. Z důvodu zpětné kompatibility jsou podporovány verze jednoduchých podtržítek z mnoha klíčových slov s dvojitým podtržítkem. Kromě toho je **__cdecl** k dispozici bez úvodní podtržítka.

Klíčové slovo **__asm** nahrazuje syntaxi jazyka C++ `asm` . `asm`je vyhrazený pro kompatibilitu s jinými implementacemi jazyka C++, ale není implementována. Použijte **__asm**.

Klíčové slovo **__based** má omezené použití pro 32 a 64 cíle kompilace.
